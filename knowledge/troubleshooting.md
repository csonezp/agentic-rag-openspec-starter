# 常见问题排查

如果真实模型调用失败，首先检查环境变量。DeepSeek 调用需要 `MODEL_PROVIDER=deepseek` 和 `DEEPSEEK_API_KEY`。如果缺少密钥，脚本应该在发起网络请求前给出中文错误提示。

如果脚本能 dry-run 但真实调用失败，需要区分网络错误、HTTP 错误和模型响应解析错误。网络错误通常发生在 DNS、代理或连接阶段。HTTP 错误通常包含状态码和服务端错误信息。解析错误说明 API 返回结构和当前 parser 预期不一致。

如果 RAG 回答不准确，不要只改 prompt。应该按顺序检查：文档是否加载到、文本是否标准化正确、chunk 是否切得合理、embedding 是否生成、top-k 是否召回相关内容、prompt 是否只包含相关证据。

如果检索总是命中错误文档，可能是文档太少、chunk 太大、术语太相似、缺少 metadata 过滤，或 query rewriting 不合适。需要用评测问题定位问题，而不是凭单次体验判断。

## 可验证事实

- 真实调用失败时先检查环境变量。
- dry-run 成功不代表真实网络调用成功。
- RAG 回答错误不一定是 prompt 问题。
- 检索问题需要结合评测问题定位。
