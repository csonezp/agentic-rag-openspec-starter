# Agent 概念边界

Agent 不是“会聊天的模型”的同义词。普通聊天机器人通常只根据当前提示词生成回答，而 Agent 会在目标、上下文、工具和反馈之间做选择。一个最小 Agent 至少应该能判断：直接回答、检索知识库、追问用户，还是调用某个受控工具。

Workflow 和 Agent 也不同。Workflow 更像预先写好的流程图，步骤固定、分支明确，适合发票处理、审批流、批量导入等稳定任务。Agent 更适合开放问题，因为它可以根据当前状态选择下一步动作。但 Agent 的自由度越高，越需要审计、权限和评测。

RAG 是 Agent 的一种常见能力，但不是 Agent 的全部。RAG 的重点是把外部知识检索出来，让模型基于证据回答；Agent 的重点是决定什么时候检索、什么时候调用工具、什么时候拒答或追问。

本项目的早期目标不是做一个全自动助手，而是先做一个可信的内部知识库 Agent。它应该能基于仓库中的知识库回答问题，给出来源引用，并在证据不足时说明无法回答。

## 可验证事实

- Chatbot 主要负责生成自然语言回答。
- Workflow 的步骤通常由开发者预先定义。
- RAG 关注检索证据和基于证据生成回答。
- Agent 关注在回答、检索、追问和工具调用之间做选择。
