# 基础 RAG 流程

基础 RAG 流程可以拆成两个阶段：索引阶段和查询阶段。索引阶段发生在用户提问之前，负责把知识源变成可检索的数据。查询阶段发生在用户提问之后，负责找到相关片段并组织回答。

索引阶段通常包括：文档加载、文本抽取和标准化、文档切片、生成 embeddings、写入向量库。文档加载只负责发现和读取来源；标准化负责把 Markdown、HTML 或 PDF 文本变成更稳定的纯文本；切片负责把长文档拆成适合检索的小块；embedding 负责把文本变成向量。

查询阶段通常包括：接收用户问题、对问题生成查询向量、检索 top-k chunks、把命中的 chunks 放入模型上下文、生成回答和来源引用。如果检索结果不足，系统应该拒答或追问，而不是让模型凭空补全。

本项目先从本地 Markdown 开始，是为了让每个步骤都可观察、可测试。后续可以替换为 PDF、HTML、飞书文档或数据库，但流程边界仍然类似。

## 可验证事实

- 文档加载发生在索引阶段。
- top-k 检索发生在查询阶段。
- 只有命中的少量 chunks 应该进入模型上下文。
- 证据不足时应该拒答或追问。
