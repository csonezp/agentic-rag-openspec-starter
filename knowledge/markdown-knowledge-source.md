# 本地 Markdown 知识源约定

本项目的第一个知识源是仓库内的 `knowledge/` 目录。这个目录只放学习用 Markdown 文档，不放真实敏感数据。选择 Markdown 是因为它容易阅读、容易版本管理，也方便在学习阶段观察加载、标准化和切片结果。

每篇 Markdown 文档应该有一个明确的一级标题。加载阶段会读取 `*.md` 文件，并保留相对路径、标题和原始内容。标准化阶段会把 Markdown 语法转换成更适合后续切片的文本，但不会改变原文件。

知识库文档应该是系统可检索的“事实材料”，不应该混入评测题和答案提示。评测问题应该放在 `evals/`，这样后续建立索引时不会把测试题当成知识内容召回。

如果后续要接入真实内部文档，需要先增加权限过滤、敏感信息处理和索引审计。本项目的学习阶段只使用模拟资料。

## 可验证事实

- 默认知识源目录是 `knowledge/`。
- 默认来源类型是 Markdown。
- 评测问题不应该放入 `knowledge/`。
- 学习阶段不放真实敏感数据。
