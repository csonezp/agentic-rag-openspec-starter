# 模型调用可观测性

模型调用可观测性用于回答三个问题：调用了什么模型、花了多久、有没有失败。没有这些信息，后续很难判断 RAG 质量问题来自检索、prompt、模型还是网络。

本项目在 Phase 1 中开始记录模型调用摘要。一次调用至少应该记录 provider、model、stream、latency_ms、status 和 error。对于 DeepSeek 这类真实 provider，还可以记录调用开始和结束时间。token 使用量如果 API 响应中提供，也应该被纳入观察对象。

可观测性记录不应该泄露密钥，也不应该默认保存完整私有文档内容。学习阶段可以打印简短摘要，生产场景则更适合写入结构化日志或 tracing 系统。

可观测性和评测不同。可观测性描述一次调用发生了什么；评测判断这次回答是否正确、引用是否准确、是否应该拒答。两者需要配合使用。

## 可验证事实

- 至少应记录 provider、model、latency 和 status。
- 日志不能泄露 API Key。
- 可观测性不是质量评测。
- token 使用量应该在 API 提供时记录。
