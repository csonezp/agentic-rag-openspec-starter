# Phase 1：LLM 应用基础

## 为什么做

Phase 0 已经建立了最小 Python 项目骨架、dry-run 模型客户端和中文文档约定。下一步需要把“模拟响应”升级为可学习、可验证的真实 LLM 应用基础能力。

这个阶段的目标不是直接做完整 RAG，而是先理解并封装模型调用的基本工程能力：请求结构、流式输出、结构化输出、工具调用和基础日志。

## 做什么

- 引入真实模型调用入口，优先围绕 OpenAI Responses API 学习。
- 保留 dry-run 能力，方便无 API Key、无网络或测试场景运行。
- 实现基础聊天调用。
- 实现流式输出示例。
- 实现一个小 schema 的结构化输出示例。
- 实现一个本地 function/tool calling 示例。
- 记录模型名、延迟、错误和基础用量信息。
- 为新增行为补充测试和运行说明。

## 不做什么

- 不在本阶段实现完整 RAG。
- 不引入复杂前端。
- 不做多 Agent 编排。
- 不接入真实业务数据。
- 不开放会修改外部状态的工具。

## 成功标准

- 没有 API Key 时，项目仍可通过 dry-run 路径运行。
- 有 API Key 时，可以运行一个真实模型调用示例。
- 可以演示一次结构化输出。
- 可以演示一次本地工具调用。
- 关键路径有测试覆盖。
- README 和学习路线文档更新为中文。
