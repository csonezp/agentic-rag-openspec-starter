# 0001：第一个知识源选择

日期：2026-04-27

## 决策

第一个知识源选择“仓库内本地 Markdown 知识库”，路径约定为 `knowledge/`。

Phase 2 开始实现基础 RAG 时，优先从 `knowledge/**/*.md` 读取文档。

## 原因

- 本地 Markdown 最容易版本化，适合学习阶段追踪变化。
- 不需要额外账号、网络、浏览器自动化或第三方 API。
- 文档结构清晰，方便学习标题层级、段落切片、来源引用和元数据提取。
- 后续可以把 PDF、网页、飞书、Notion 或 Confluence 作为扩展来源逐步加入。

## 影响

- Phase 2 的文档加载器先实现 Markdown 文件读取。
- 初始元数据至少包含 `source_path`、`title`、`heading_path` 和段落位置。
- RAG 回答中的引用先指向本地文件路径和标题层级。
- 暂不处理权限隔离；真实私有数据接入前再补文档级权限控制。

## 替代方案

- PDF：更贴近真实知识库，但解析噪音更大，不适合作为第一步。
- 网页：可以练习抓取和清洗，但会引入网络和页面结构不稳定因素。
- 飞书/Notion/Confluence：更接近企业场景，但认证和权限会让学习路径变重。
- OpenAI 托管 file search：上手快，但不利于先理解 RAG 的切片、检索和评测细节。
